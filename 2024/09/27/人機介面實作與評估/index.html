人機系統之基本概念

Part1 人機互動之人因設計基礎 設計：「問題」～「解決」的過程

設計Design：問題 -> 解決
新點子放入產品，並考慮可行性。

設計專家：系統性方法地進行。
浪漫設計師：自己想法最優先，成本高，耗時。

設計跟藝術不同，有特別程序與方法(系統與經驗)。

推薦書籍：物生物-現代設計理念

P(問題，描述問題，先不要提出解法) -> DP(說明問題，方法分析) -> 分解成單元 -> RD(收集資訊) -> AD(分析資訊) -> C(創見) -> I(理念) -> MT(材料與科技) -> SP(實驗) -> M(製成的模型與樣品) -> V(考驗) -> S(解決問題)

破解問題方法之一：Fishbone Diagram

了解問題所在，然後解決(可能有多種方法)

認真探索問題內涵，試著用不同角度思考問題。

HCD(Human Centered Design)：為人解決問題(方法論)

UCD(User Centered Design)：為人設計中心過程策略 1. 規劃設計開發流程 2. 量測系統與介面成效 3. 調整系統可適化 4. 進行設計 5. 進行使用者研究

Q(思考)：學設計初衷？
A：學習新事物，獲得有趣的課外知識。

Part2 人機介面實作與評估

人為失誤類型

  • 取代錯誤(substitution errors)：誤認A為B
  • 調整錯誤(adjustment errors)：過與不及
  • 遺忘失誤(reversal errors)：方向錯亂
  • 無意啟動(unintentional activation)：不該動的去動了…
  • 無法搆及(inability to reach)：太遠了

人因觀點

科學發現、產業研究、人類順從 -> 人們構思、科學研究、科技順從、使人適事 -> 適事適人

人性化設計(人因工程)

  • 強調實徵資料(Empirical)和評估(evaluation)
  • 科學方法、客觀數據與假設，人類行為資料。
  • 系統導向(system orientation)(產品 環境 程序 人)

課中影片：What is Retina Display?!

人因工程

  • 人與週邊交互
  • 目標：效果(effectiveness)和效率(efficiency)
  • 人的因素用於設計中
    機具導向時期(machine oriented)：工作研究、工廠管理(fitting job to person) -> 人員導向時期(man oriented) -> 系統導向時期(system oriented)：受到重視

不是普通知識(common sense)，必須反覆測試，止於checklist和guideline運用，配合經驗法則

人因工程議題

  • 人因工程基本知識
    • 感覺系統
    • 輸入與輸出
  • 人機介面與系統
    • 顯示與控制
  • 人體測計

    • 平均人與平均設計(average)

      • 極端設計與可調設計，不可以以平均尺寸作為設計基準
  • 極端設計(extreme)
    • 使母群體大部分能適合的設計
  • 可調設計(adjustable)
    • 適合各種體型需求，需考慮成本

感覺系統與顯示介面設計

人機互動之人因設計基礎 人體感覺系統基礎概念 Part3

  • 感知(訊息接收)(Sensing: Information Receiving)
  • 訊息儲存(Information Storage)
  • 訊息處理與決策(Information Processing & Decision)
  • 動作功能(Action Functions)
  • 絕對閾(absolute threshold)：引起任何感覺之前，接受刺激必須達某一最低強度(下絕對閾)，上升到感受器喪失知覺(上絕對閾)
  • 差異閾(differential threshold)：能夠感受兩刺激間有所差異的最小差異量
  • 視銳度(visual acuity)：視力
  • 辨色力(color discrimination)：紅/綠
  • 暗適應力(dark adaptation)：暗適應時間>亮適應時間
  • 視覺
    • 亮度對比
    • 照明水準
    • 觀看時間
    • 眩光
    • 運動
  • 聽覺
    • 遮蔽
    • 遮蔽效應：有無遮蔽下絕對閾之差，車速正比音量
  • 嗅覺
  • 觸覺
    • 兩點覺閾(twopoint threshold)：關係到觸覺顯示器辨別，溫度正比於觸覺敏銳度
  • 運動覺
  • 平衡覺
